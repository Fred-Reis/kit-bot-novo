## Phase 4 — Bot Admin Endpoints

Implement on bot side, then replace MSW write handlers in web.

### Task 4.1 — Bot: JWT auth middleware

`apps/bot/src/middleware/auth.ts`:
- Extract `Authorization: Bearer <token>` from request
- Call `supabase.auth.getUser(token)` to validate
- Attach `ownerId` to request context
- Return 401 if invalid

**Acceptance:** valid JWT passes; invalid JWT → 401.

### Task 4.2 — Bot: approve-kyc endpoint

`POST /admin/leads/:id/approve-kyc`
- Validate JWT (middleware)
- Update `Conversation.data.kycApproved = true`
- If `context.residents === 1` → set state to `residents_docs_complete`
- Else → send WA message to lead requesting residents' docs
- Notify owner via WhatsApp that KYC approved

Remove MSW handler in web.

**Acceptance:** calling endpoint updates lead state in DB; lead receives WA message.

### Task 4.4 — Bot: confirm-payment endpoint (Phase 1.5 — deferred)
